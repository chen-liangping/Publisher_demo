# MongoDB 数据库权限管理需求文档

## 1. 功能概述

MongoDB 数据库权限管理功能为用户提供了完整的数据库权限管理解决方案，包括用户管理、数据库管理和查询工具。该功能简化了 MongoDB 的权限管理复杂性，让用户能够直观地管理数据库访问权限。

## 2. 核心功能

### 2.1 双用户权限模式
- **只读用户**：仅具有数据查询权限，适用于数据分析和报表场景
- **读写用户**：具有完整的数据操作权限，适用于应用程序数据操作

### 2.2 数据库权限管理
- **权限可视化**：通过表格形式清晰展示每个数据库的权限状态
- **一键权限控制**：点击权限标签即可授予/撤销权限
- **自动权限授予**：新建数据库时自动为两个用户授权

### 2.3 数据库查询工具
- **专业查询界面**：提供类似 MongoDB Shell 的查询环境
- **实时查询执行**：支持 MongoDB 查询语句的实时执行
- **结果可视化**：以友好的格式展示查询结果

## 3. 功能详细说明

### 3.1 用户信息展示

#### 功能描述
在 MongoDB 实例卡片中显示双用户信息，区别于其他数据库类型的单用户模式。

#### 显示内容
- **只读用户**：用户名和密码（支持一键复制）
- **读写用户**：用户名和密码（支持一键复制）
- **实例信息**：版本、架构类型、连接数、端口等

#### 技术实现
- 条件渲染：仅对 MongoDB 类型显示双用户布局
- 安全显示：密码采用掩码显示，支持复制功能
- 响应式布局：适配不同屏幕尺寸

### 3.2 权限管理界面

#### 功能描述
提供专门的权限管理弹窗，以数据库为中心进行权限管理。

#### 界面布局
```
┌─────────────────────────────────────────┐
│ MongoDB 权限管理 - [实例名称]           │
├─────────────────────────────────────────┤
│ 实例信息区域                            │
│ - 图标 + 实例名称 + 规格信息            │
│ - 账号信息：只读账号 | 读写账号         │
│ - [新建数据库] 按钮                     │
├─────────────────────────────────────────┤
│ 数据库权限管理表格                      │
│ ┌─────────────┬──────────┬────────────┐ │
│ │ 数据库名称  │ 用途备注 │ 权限状态   │ │
│ ├─────────────┼──────────┼────────────┤ │
│ │ gamedata    │ 游戏数据 │ [已授权 ✓] │ │
│ │ userinfo    │ 用户信息 │ [已授权 ✓] │ │
│ └─────────────┴──────────┴────────────┘ │
├─────────────────────────────────────────┤
│ 说明：新建数据库后自动授权给两个账号    │
└─────────────────────────────────────────┘
```

#### 核心特性
- **统一权限状态**：所有数据库默认为已授权状态
- **备注管理**：支持为每个数据库添加用途说明
- **简化操作**：无需复杂的权限配置，专注于数据库管理

### 3.3 新建数据库功能

#### 功能描述
提供简洁的数据库创建流程，自动处理权限分配。

#### 创建流程
1. 点击"新建数据库"按钮
2. 填写数据库名称（必填）
3. 填写用途备注（可选，最多200字符）
4. 系统自动创建并授权

#### 自动化处理
- **权限自动授予**：创建后自动为只读和读写用户授权
- **状态同步**：权限管理界面实时更新
- **备注保存**：用途备注与数据库关联保存

### 3.4 数据库查询工具

#### 功能描述
提供专业的 MongoDB 查询环境，支持实时查询执行。

#### 界面设计
```

│ 数据库查询                        [×]   │
├─────────────────────────────────────────┤
│ ● MongoDB                               │
│ 为测试环境，仅支持查询，请勿修改数据    │
├─────────────────────────────────────────┤
│ ┌─────────────────────────────────────┐ │
│ │ > use testapp                       │ │
│ │                                     │ │
│ │                                     │ │
│ │                                     │ │
│ └─────────────────────────────────────┘ │
│ Ctrl+Enter 执行查询        [执行查询]   │
├─────────────────────────────────────────┤
│ 查询结果                                │
│ ┌─────────────────────────────────────┐ │
│ │           暂无内容                  │ │

│ └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

#### 技术特性
- **专业编辑器**：深色主题，等宽字体，类似 MongoDB Shell
- **快捷键支持**：Ctrl+Enter (Mac: Cmd+Enter) 执行查询
- **结果展示**：JSON 格式化显示查询结果
- **安全限制**：仅支持查询操作，禁止数据修改

## 6. 用户体验设计

### 6.1 设计原则
- **简化复杂性**：将复杂的 MongoDB 权限管理简化为直观的界面操作
- **一致性**：与其他数据库类型保持界面风格一致
- **专业性**：查询工具提供专业的开发者体验

### 6.2 交互特色
- **视觉反馈**：权限状态通过颜色和图标清晰表达
- **操作便捷**：一键复制、快捷键执行等提升操作效率
- **信息完整**：提供充分的上下文信息和操作说明

## 7. 技术实现要点

### 7.1 条件渲染
- MongoDB 类型使用特殊的双用户显示布局
- 其他数据库类型保持原有的单用户显示

### 7.2 状态管理
- 使用 React Hooks 管理弹窗状态、表单数据
- 实时同步权限变更和数据库创建

### 7.3 安全考虑
- 密码掩码显示，防止敏感信息泄露
- 查询工具限制为只读操作
- 输入验证和错误处理
