# 2. 需求概述

## 2.1 需求简介

在 Publisher 平台支持将**测试环境**配置与数据**同步/迁移**至**生产环境**，提供**全量同步**与**部分迁移**两种模式；部分迁移需可选客户端（版本/配置/CDN/跨域）与服务端（应用、镜像、共享文件、存储、告警联系人）数据项，并在执行前确认、执行中展示各组件进度与日志。

## 2.2 范围、非目标与前提假设

**范围（本需求必须覆盖）**

- 在「当前游戏/当前项目已选定测试环境、可切换至生产」的前提下，提供测试→生产的同步向导与执行反馈。
- 危险操作的前置提示、确认页、步骤内状态与日志展示（与后端能力匹配部分）。

**非目标（不默认包含，若要做需另开需求）**

- 跨项目、跨游戏的一次性批量同步；生产→测试回灌；定时自动同步。
- 与同步无关的**生产环境容量规划、成本**计算。
- 除 Web 管理台外的 **OpenAPI/CLI** 同步能力（若需要，单独立项）。

**前提假设（须与业务确认，【待决】可入附录 B）**

- 测试与生产为**一一对应**的「环境对」（同一 `gameId`/项目下）；若存在一对多/多对一，须先补充**环境映射**产品规则（见 2.2.1）。

### 2.2.1 环境映射（待业务确认）

- 默认：当前 UI 所选的**一个测试环境**仅同步到**一个**已绑定的生产环境；映射关系以平台配置为准。
- 若未来支持多生产目标：需增加「目标环境」选择步或全局设置，并单独定义冲突与审计规则；**未确认前**按单目标实现。

## 2.3 风险点

- 同步为**不可逆**写操作，误选范围可能造成生产配置被覆盖。
- 执行顺序、失败重试、初始化空态等与《测试环境上生产流程》或后端实现可能存在差异，上线前需与后端契约对齐（见文末差异清单）。

## 2.4 设计稿

- **【待决】**正式设计稿；**UI/UX 联调**前需具备：多步骤向导、双栏数据选择、步骤 3 大段日志/状态的低保真或高保真，以满足危险操作可评审、可测。

**端与可访问性（产品口径）**

- 默认按 **PC Web** 管理台设计；**是否支持移动端/窄屏**需明确：若不支持，在帮助文案中说明「请使用桌面浏览器」。
- 长列表、多步骤：关键操作（确认、开始同步）**焦点与键盘路径**在联调时抽查（不必单独成项，可写入验收表一条）。

## 2.5 角色、权限与操作审计

**角色/权限（待业务与账号体系定稿）**

- 建议至少区分：**可发起同步**、**仅查看**、**无权限**（不展示入口或灰显+说明）。
- 「同步数据」与「写生产类」其他能力是否**同一权限点**：需与权限中台统一，避免入口可见但提交 403（或相反）。

**审计（建议作为发布前必选项）**

- 记录：操作人、时间、源环境、目标环境、同步模式（全量/部分）、**已选范围摘要**（可结构化 ID 或哈希，避免过大）、任务 ID、**终态**（成功/部分失败/失败/中断）。
- 展示：管理台「操作记录/审计」或仅后端留存需产品拍板；**保留周期**与合规要求待法务/安全确认。

## 2.6 「不可逆」与部分失败的对外说法

- **不可逆**：指已写入生产的变更**无产品化一键回滚**；不表示「任一步失败则整单未发生任何写」。
- **部分成功**：须在前端与结果页明确：哪些组件成功、哪些失败；客服/用户可见的**标准话术**由产品与运营补一段（避免与 4.1 矛盾）。

# 3. 功能详情

## 3.1 入口与触发

### 3.1.1 入口

- 主界面测试环境切换到正式环境时，提供「**同步数据**」，点击后打开「**数据同步**」弹窗。

### 3.1.2 弹窗基础

| 项目 | 说明 |
|------|------|
| 标题 | 数据同步 |
| 宽度 | 约 1000px（以设计稿为准） |
| 遮罩关闭 | 不允许点击遮罩关闭，降低误操作 |
| 关闭后状态 | 关闭弹窗后应结束本次向导流程（再次进入从第一步开始） |

## 3.2 前置条件与校验

### 3.2.1 业务规则（产品侧）

- 前置条件：**测试环境已完成初始化**。
- **「测试环境初始化完成」** 须为**可机读**判定（如接口状态位、必备元数据就绪标志等），**不得**仅依赖用户主观理解；**具体字段/接口**与后端在联调前写入接口契约。与 **3.9 生产环境初始化**、**4.2 生产未就绪** 的禁用/引导规则**分条定义**，不得混为同一句文案。

### 3.2.2 已实现提示

- 选择同步类型步骤底部 **重要提示**（警告样式）：同步不可回退、需确认生产已初始化。

## 3.3 同步类型与步骤流

### 3.3.1 步骤条（固定 4 步）

| 序号 | 标题 | 说明 |
|------|------|------|
| 0 | 选择同步类型 | 全量同步或部分迁移 |
| 1 | 选择数据 | 部分迁移时展示；全量跳过 |
| 2 | 确认配置 | 二次确认 |
| 3 | 同步进度 | 展示进度与日志 |

### 3.3.2 全量同步

- **交互**：步骤 0 选择「全量同步」后，**直接进入确认步骤**（步骤 2），跳过数据勾选。
- **确认页文案**：测试环境所有数据同步至生产，包含版本文件、配置文件、CDN 配置、源站配置、镜像文件、应用、存储实例、共享文件、告警联系人；同步后不可回退。

### 3.3.3 部分迁移

- **交互**：步骤 0 选择「部分迁移」→ 步骤 1 勾选范围 → 步骤 2 确认 → 步骤 3 执行。

### 3.3.4 切换同步类型时的状态重置

- 在步骤 0 从「部分迁移」与「**全量同步**」**互相切换**时，须**清空**步骤 1 已选客户端/服务端范围及「迁移后名称」等**仅本次向导有效**的临时态，再进入对应空态/确认页，避免全量与部分选集混用。
- 可与「关闭弹窗后从第一步重进」**复用同一套 reset 逻辑**（实现约定）。

## 3.4 部分迁移 — 客户端数据

**布局**：左侧区域标题「客户端数据」；区域右上角支持一键全选/全不选（版本文件 + 配置文件 + CDN 两项 + 跨域同时满足才算「全选」态）。

### 3.4.1 版本文件

| 项目 | 说明 |
|------|------|
| 展示 | 多选列表，选项来自测试环境已发布的版本文件 |
| 折叠 | 支持展开/收起列表；**建议**客户端各子块折叠状态相互独立（避免与右侧区域共用一套展开状态） |
| 分段全选 | 支持仅「版本文件」维度全选/全不选 |

### 3.4.2 配置文件

- 与版本文件相同：多选文件列表；支持分段全选/全不选。

### 3.4.3 CDN 配置

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| CDN 源站配置 | 是否 | 否 | 独立勾选项 |
| CDN 缓存配置 | 是否 | 否 | 独立勾选项 |
| 一键 CDN 全选 | — | — | 仅勾选上述两项；**不包含**跨域 |

### 3.4.4 跨域配置

| 字段 | 类型 | 默认值 |
|------|------|--------|
| 跨域配置 | 是否 | 否 |


## 3.5 部分迁移 — 服务端数据

**布局**：右侧区域标题「服务端数据」；一键全选条件为：应用数、镜像数、共享文件数、存储数均等于测试环境全量（**不包含**告警联系人勾选）。

### 3.5.1 应用

| 项目 | 说明 |
|------|------|
| 选择 | 多选；每项展示应用名称 + 分类标签（如游服/平台/测试等，样式区分） |
| 迁移后名称 | 选中后出现单行输入，占位「迁移后名称」；默认填入应用展示名，若无则填应用别名 |
| 提示 | 「应用同步将同步应用所有数据，包含应用类型、部署方式、容器配置、镜像、标签、插件、挂载的文件等应用配置」 |

**业务规则**

- 勾选应用时：**自动勾选**该应用已挂载的镜像、共享文件（并入当前选择）。
- 取消应用时：从已选镜像/文件中移除**不再被其余已选应用依赖**的项；对策略允许单独取消的资源，可保留在用户选择集中。

### 3.5.2 镜像

| 项目 | 说明 |
|------|------|
| 列表 | 展示镜像名；可选中/取消（受规则限制） |
| 不可取消 | 被**当前已选应用**挂载且策略为「必选同步」的镜像：默认勾选且不可取消 |
| 标签 | 被已选应用挂载时展示「已挂载」 |
| 名称 | 镜像名称**不可修改**（不提供迁移后别名输入） |

### 3.5.3 共享文件

- 规则同镜像：被已选应用挂载且为必选时不可取消；其余文件可选中或取消。

### 3.5.4 存储

| 项目 | 说明 |
|------|------|
| 选择 | 多选 |
| 迁移后名称 | 选中后出现单行输入；默认填入存储别名 |
| 提示 | 同步存储类型、引擎版本、实例类型、账号密码等信息，且不可回退 |

### 3.5.5 告警联系人（值班用户）

| 项目 | 说明 |
|------|------|
| 总开关 | 「同步所有告警联系人」 |
| 勾选后 | 列出将同步的联系人（姓名、AppId、DingTalkId） |
| 产品规则 | 按 AppId、姓名、DingTalkId 判断是否已存在；不存在则创建——**待后端** |

### 3.5.6 迁移对比预览

- **出现条件**：已选应用/镜像/文件/存储任一非空，或开启告警联系人同步。
- **表格列**：数据类型、测试环境（迁移前）、生产环境（迁移后）；生产列与测试不一致时高亮并标注「已修改」。
- **行范围**：应用、镜像、共享文件、存储、告警联系人。镜像与共享文件在表中「生产」列可与测试同名展示；**支持修改展示名/别名**的为应用、存储。

### 3.5.7 命名冲突与校验（应用 / 存储的「迁移后名称」）

| 项目 | 要求 |
|------|------|
| 与生产现网**重名** | 在可提交执行前**须阻断并提示**；若产品允许**覆盖/合并**，须**单独定义**策略、二次确认与审计，并写入接口契约。**禁止**静默覆盖。 |
| 字符集与长度 | 与生产**命名规范**一致；不合法时内联错误，**不得**进入步骤 3 执行。 |
| 大小写与唯一性 | 与生产现网唯一性规则一致，并在界面上有明确说明（如是否大小写不敏感）。 |
| 单条失败时的任务策略 | 整单失败 vs 跳过该项继续：**须与 4.1 一致**并由产品与后端在【待决】中定稿，见附录 B。 |

### 3.5.8 「必选同步」策略来源

- 「被已选应用挂载且**必选同步**」的规则来源（配置台 / 元数据 / 内置枚举）：**【待决】** 与权威配置说明或接口文档**对齐**；**附录 B** 跟踪定稿后引用路径。

## 3.6 确认页（步骤 2）

### 3.6.1 全量

- 仅展示全量确认提示（见 3.3.2）。

### 3.6.2 部分迁移

- 提示：已选数据、不可回退。
- 「同步内容概览」：左列列出已选版本文件、配置文件、CDN/跨域项；右列汇总应用/镜像/共享文件/存储数量及告警联系人数。

### 3.6.3 底部强提示

- 文案：「同步一旦开始不可中止，请确认所有选择无误」（与主按钮「确认并开始同步」并存）。

## 3.7 执行与进度（步骤 3）

### 3.7.1 触发

- 主操作「**确认并开始同步**」（强调危险操作的主按钮样式，配同步类图标）；点击后进入同步进度步骤，并进入执行态（进度区展示）。

### 3.7.2 进度组件顺序

以下顺序与产品文档「服务端：应用 → 镜像 → 共享文件 → …」**不一致**，当前统一流水线顺序为：

镜像 → 共享文件 → 应用 → 存储 → 告警联系人 → 版本文件 → 配置文件 → CDN 配置

**说明**：若后端必须采用文档顺序，需在联调单中列为**实现与文档差异**并协调界面或接口约定。

### 3.7.3 状态与展示

| 状态 | 展示 |
|------|------|
| 等待中 | 占位样式 + 文案「等待中」 |
| 进行中 | 加载动效 + 强调背景 + 文案「进行中」 |
| 已完成 | 成功态图标 + 文案「已完成」 |
| 失败 | 失败态图标 + 文案「失败」 |

- 每一类数据同步块内展示**带时间的明细日志**（成功/失败及原因说明）；失败时日志区允许更高展示，便于阅读。
- 总进度：按「已完成 + 已失败」的组件数占全部组件数的比例展示。

### 3.7.4 完成提示

- 全部阶段结束后给予**全局结果反馈**（全部成功 / 部分失败）。界面提示须与**后台异步任务终态**一致（轮询或推送）；**具体轮询间隔、超时与重连** 见接口契约、**【待决】** 入附录 B。

## 3.8 取消与返回

| 场景 | 行为 |
|------|------|
| 弹窗取消或关闭，且**未在同步中** | 清空本次向导状态并关闭 |
| **同步中** 用户发起关闭/取消 | 均须**二次确认**；确认后关闭弹窗；取消则留在当前步。**不得**用点击遮罩关闭。 |
| 步骤 1 / 2 | 底部「上一步」返回上一步 |
| 步骤 3（同步中）与 3.6.3 的关系 | **【待决，附录 B 必须关闭其一】** **方案 A**（与 3.6.3 字面一致：过程不可人工作废）：步骤 3 内不提供「停止同步」；仅结束**本弹窗的展示**（重进后可从任务中心看进度/结果，**若** V1 有任务中心）。**方案 B**（可请求中止任务）：显式提供「停止」并**同步改 3.6.3 文案**；**是否**能真正杀后台任务**以后端能力为准**。V1 发布前在附录 B 勾选一个方案并定稿 3.6.3/3.8/接口。 |
| 步骤 3 的重复操作 | 同步中**不**再提供可点的「再次开始」主流程；与 3.7.1 触发一次相区分。 |

## 3.9 初始化正式环境（原 3.4 节能力）

- 原需求：管理台配置、展示初始化内容、刷新后按最新配置初始化；生产配置项仅能从测试新增不可删减。
- **当前版本**：未实现对应页面与接口，整节标为 **待实现**。

# 4. 异常情况处理

## 4.1 同步执行异常

- 某组件同步失败时是否**继续**后续组件：**待后端**与产品确认；若允许继续，须在进度列表与日志中区分成功与失败项，并与重试策略、部分回滚策略一致。
- 网络/权限/配额等错误分类与界面展示文案 → **待后端**定义错误码与映射。

## 4.2 前置与权限异常

- 生产未初始化 → 当前仅文案提示；**待实现**：禁用同步或引导「初始化正式环境」。
- 测试/生产侧未初始化无可迁数据 → **待实现**：卡片空态「无可迁移数据」。

## 4.3 通用兜底

- 禁止点击遮罩关闭，降低误关；用户仍可通过显式「取消」关闭。
- 与后端契约不一致的字段/顺序/重试 → 联调前在本文档附录清单中维护。

## 4.4 日志与敏感信息

- 进度、明细及可导出内容中，**不得**以明文展示存储密码、Token、完整连接串、Cookie 等敏感信息；须掩码、占位（如「已配置」）或**末几位**等展示方式；**【待决：具体掩码规则】** 经安全/后端评审后，写入发布清单或接口说明。
- 对客服/导出的**同等脱敏**要求写进测试验收（见第 6 节可增一条）。

# 5. 非功能与体验（评审必补）

## 5.1 任务时长、超时与离开页面

| 项目 | 建议 |
|------|------|
| 预期时长 | 全量/部分迁分别给出**数量级**（如分钟级/小时级），供预期管理；**硬超时**由后端定义，前端展示「已超时，请重试/联系支持」等。 |
| 用户离开弹窗/刷新页面 | **推荐**：以 `taskId` 为维度的**异步任务**；提供「**任务中心** / 同入口查看进度」；若 V1 仅弹窗内轮询，则须在文档中写明「关闭弹窗**不保证**能恢复同一任务」或「通过任务中心恢复」二选一。 |
| 长任务中 Token 过期 | 轮询/推送 401/403 时：统一提示重新登录，并说明任务是否**仍在后端执行**（**待后端**）。 |

## 5.2 并发与重复提交

| 项目 | 建议 |
|------|------|
| 同一项目多用户同时发起 | **产品定一策**：禁第二单 / 排队 / 仅只读看他人任务；与后端**幂等 key**、任务锁一致。 |
| 连点「确认并开始」 | 前端**防抖+提交后禁用**；后端**幂等**（同一任务体_hash 不重复建单），避免双写。 |
