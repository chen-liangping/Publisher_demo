## 系统概述

### 功能简介
负载均衡系统用于管理负载均衡实例、配置监听规则、转发策略、虚拟服务器组和SSL证书，实现流量分发和高可用性。

### 主要功能模块
- 负载均衡实例管理
- 监听规则配置
- 转发策略管理
- 虚拟服务器组管理
- SSL证书管理

## 2. 负载均衡详情页

### 2.1 监听规则 Tab

#### 2.1.1 展示字段

| 字段名称 | 字段标识 | 字段类型 | 字段含义 | 备注 |
|---------|---------|---------|---------|------|
| 规则名称 | name | String | 监听规则的标识名称 | 唯一标识 |
| 协议 | protocol | Enum | 监听协议类型 | HTTP/HTTPS/TCP/UDP |
| 端口 | port | Number | 监听端口号 | 1-65535 |
| SSL证书 | certificate | String | 绑定的SSL证书 | 仅HTTPS协议显示 |
| 服务器组 | serverGroup | String | 后端虚拟服务器组名称 | 必须关联 |
| 健康检查 | healthCheck | Enum | 健康检查状态 | enabled/disabled |
| 状态 | status | Enum | 监听规则状态 | active/inactive |

#### 2.1.2 新增/编辑监听规则

**需要填写的字段：**

| 字段名称 | 字段标识 | 字段类型 | 是否必填 | 字段规则 | 默认值 | 字段说明 |
|---------|---------|---------|---------|---------|-------|---------|
| 规则名称 | name | String | 是 | 只能包含字母、数字、下划线<br>必须以字母或下划线开头<br>正则：`^[a-zA-Z_][a-zA-Z0-9_]*$` | 无 | 监听规则的唯一标识符 |
| 协议 | protocol | Enum | 是 | HTTP / HTTPS / TCP / UDP | HTTP | 监听的网络协议类型 |
| 端口 | port | Number | 是 | 1-65535<br>整数 | 无 | 监听的端口号 |
| SSL证书 | certificate | String | 条件必填 | 当协议为HTTPS时必填 | 无 | 选择已上传的SSL证书 |
| 后端服务器类型 | - | String | - | 固定值："虚拟服务器组" | 虚拟服务器组 | 不可修改 |
| 虚拟服务器组 | serverGroup | String | 是 | 从已创建的服务器组中选择 | 无 | 流量转发的目标服务器组 |
| 健康检查 | healthCheck | Enum | 是 | enabled / disabled | enabled | 是否启用后端服务器健康检查 |

**字段联动规则：**
- 当"协议"选择"HTTPS"时，显示"SSL证书"字段（必填）
- 当"协议"选择其他值时，隐藏"SSL证书"字段
- 选择"虚拟服务器组"后，自动显示该组包含的虚拟机信息

#### 2.1.3 配置转发策略

**触发方式：** 点击监听规则操作列的"配置转发策略"按钮

**需要填写的字段：**

| 字段名称 | 字段标识 | 字段类型 | 是否必填 | 字段规则 | 字段说明 |
|---------|---------|---------|---------|---------|---------|
| 域名 | domain | String | 是 | **泛解析域名：**<br>- 格式：`*.test.com`<br>- * 必须在第一个字符<br>- 格式必须是 `*.xxx.xxx` 或 `*aaa.xxx.xxx`<br>- * 不能在最后<br><br>**标准域名：**<br>- 格式：`www.test.com`<br><br>正则：`/^(\*\.[\w-]+\.[\w-]+\|[\w-]+\.[\w-]+(\.[a-z]+)*)$/` | 转发规则的域名匹配条件 |
| URL路径 | path | String | 否 | - 长度限制：2-80个字符<br>- 只能使用字母、数字和 `-/.%?#&`<br>- 不能只为 `/`<br>- 必须以 `/` 开头<br>- 正则：`/^\/[a-zA-Z0-9\-\/.%?#&]{1,79}$/`<br>- **为空表示转发所有路径** | URL路径匹配规则，支持通配符 |
| 虚拟服务器组 | serverGroup | String | 是 | 从已创建的服务器组中选择 | 匹配该规则的流量转发目标 |
| 备注 | remark | String | 否 | 无限制 | 转发规则的说明信息 |

**表单结构说明：**
- 支持多个域名组
- 每个域名组包含多条路径规则
- 每条路径规则包含：URL路径 + 虚拟服务器组 + 备注
- 同一域名可以配置多条不同路径的规则

**操作说明：**
- **添加域名**：点击"添加域名"按钮，新增一个域名组
- **添加规则**：在域名组内点击"添加规则"按钮，为该域名新增路径规则
- **删除规则**：点击路径规则行的"删除"按钮
- **删除域名组**：点击域名输入框后的"X"按钮
- **添加到列表**：验证通过后，将所有配置的规则添加到转发策略列表
  - 按钮在有验证错误时显示为红色且禁用
  - 验证项包括：域名必填、域名格式、至少一个服务器组、URL格式（如果填写）

**验证提示：**
- 域名格式错误时，输入框显示红色边框，下方显示"域名格式不正确"
- URL格式错误时，输入框显示红色边框，下方显示具体错误信息
- 提交时验证：每个域名至少需要选择一个虚拟服务器组

#### 2.2.2 新增/编辑虚拟服务器组

**页面模式：** 全页面展示（非弹窗）

**需要填写的字段：**

##### 第一部分：服务器组名称

| 字段名称 | 字段标识 | 字段类型 | 是否必填 | 字段规则 | 字段说明 |
|---------|---------|---------|---------|---------|---------|
| 服务器组名称 | name | String | 是 | - 长度限制：1-80个字符<br>- 支持字符：中文、字母、数字、短划线(-)、正斜线(/)、半角句号(.)、下划线(_)<br>- 正则：`/^[\u4e00-\u9fa5a-zA-Z0-9\-\/.\\_]+$/` | 服务器组的唯一标识名称 |

##### 第二部分：选择虚拟机

**展示方式：** 表格形式，支持多选

**虚拟机信息字段：**

| 字段名称 | 字段标识 | 字段类型 | 字段含义 |
|---------|---------|---------|---------|
| 虚拟机名称 | name | String | 虚拟机的名称 |
| 内网IP | privateIp | String | 虚拟机的内网IP地址 |

**选择规则：**
- 必须至少选择1台虚拟机
- 支持多选
- 已选中的虚拟机在配置区域自动显示

##### 第三部分：配置端口和权重

**触发条件：** 在第二部分选择虚拟机后，配置区域自动出现在下方

**每个虚拟机的配置字段：**

| 字段名称 | 字段标识 | 字段类型 | 是否必填 | 字段规则 | 默认值 | 字段说明 |
|---------|---------|---------|---------|---------|-------|---------|
| 端口 | port | Number | 是 | 1-65535<br>整数 | 8080 | 虚拟机的监听端口 |
| 权重 | weight | Number | 是 | 0-100<br>整数 | 100 | 流量分配权重，数值越大流量越多 |

**配置规则：**
- 一个虚拟机可以配置多个端口
- 每个端口需要单独配置权重
- 默认每个虚拟机有一个端口配置（端口8080，权重100）
- 支持动态添加/删除端口配置

**页面操作：**
- **添加端口**：点击虚拟机卡片内的"+ 添加端口"按钮
- **删除端口**：点击端口配置行的"删除"按钮（至少保留1个端口）
- **移除虚拟机**：点击虚拟机卡片右上角的"移除虚拟机"按钮
  - 同时从选择列表中取消勾选
  - 删除该虚拟机的所有配置

**提交条件：**
- 服务器组名称已填写且格式正确
- 至少选择了1台虚拟机
- 每台虚拟机至少有1个端口配置
- 所有端口和权重值在有效范围内

### 3.1 展示字段

|---------|---------|---------|---------|------|

### 3.2 上传证书

**需要填写的字段：**

| 字段名称 | 字段标识 | 字段类型 | 是否必填 | 字段规则 | 字段说明 |
|---------|---------|---------|---------|---------|---------|
| 证书名称 | name | String | 是 | - 长度限制：2-128个字符 | 证书的标识名称，建议使用域名 |
| 证书类型 | type | Enum | 是 | server / ca | 服务器证书或CA证书 |

**服务器证书（type=server）专用字段：**

| 字段名称 | 字段标识 | 字段类型 | 是否必填 | 字段规则 | 字段说明 |
|---------|---------|---------|---------|---------|---------|
| 公钥证书 | publicKey | String | 是 | - 必须以 `-----BEGIN CERTIFICATE-----` 开头<br>- PEM格式<br>- 正则：`/^-----BEGIN CERTIFICATE-----/` | 服务器的公钥证书内容 |
| 私钥 | privateKey | String | 是 | - 必须以 `-----BEGIN PRIVATE KEY-----` 或<br>`-----BEGIN RSA PRIVATE KEY-----` 开头<br>- PEM格式<br>- 正则：`/^-----BEGIN (RSA )?PRIVATE KEY-----/` | 服务器的私钥内容 |

**CA证书（type=ca）专用字段：**

| 字段名称 | 字段标识 | 字段类型 | 是否必填 | 字段规则 | 字段说明 |
|---------|---------|---------|---------|---------|---------|
| 客户端CA公钥证书 | caPublicKey | String | 是 | - 必须以 `-----BEGIN CERTIFICATE-----` 开头<br>- PEM格式<br>- 正则：`/^-----BEGIN CERTIFICATE-----/` | 客户端CA公钥证书内容 |

**字段联动规则：**
- 选择"服务器证书"时，显示"公钥证书"和"私钥"字段，隐藏"客户端CA公钥证书"
- 选择"CA证书"时，显示"客户端CA公钥证书"字段，隐藏"公钥证书"和"私钥"
- 切换证书类型时，自动清空另一类型的字段值

**输入框特性：**
- TextArea多行输入，8行高度
- 使用等宽字体（monospace）便于查看证书内容
- 小字体（12px）

### 3.3 证书操作限制

**删除限制：**
- 已关联监听规则的证书不可删除
- 删除按钮在有关联时显示为禁用状态
- 尝试删除时提示："该证书已关联监听规则，无法删除"

## 业务逻辑

### 1. 负载均衡实例管理

#### 1.1 创建流程
1. 用户点击"创建负载均衡"按钮
2. 输入负载均衡名称
3. 系统自动配置默认参数（网络类型、规格、VPC、带宽等）
4. 生成唯一ID（格式：`lb-bp` + 时间戳）
5. 初始状态为"配置中"
6. 显示成功提示
7. 新实例显示在列表中

#### 1.2 删除流程
1. 用户点击"删除"按钮
2. 弹出二次确认对话框
3. 确认后从列表中移除
4. 显示删除成功提示

**注意：** 实际生产环境应检查是否有关联资源（监听规则、服务器组等）

### 2. 监听规则管理

#### 2.1 新增监听规则流程
1. 点击"新增监听规则"按钮
2. 填写规则名称、协议、端口
3. 如果协议选择HTTPS，必须选择SSL证书
4. 选择虚拟服务器组
5. 自动显示该服务器组包含的虚拟机信息（名称+IP）
6. 设置健康检查状态（默认启用）
7. 验证通过后保存
8. 生成唯一ID（格式：`listener-` + 时间戳）
9. 状态默认为"active"
10. 列表自动刷新

#### 2.2 编辑监听规则流程
1. 点击操作列的"编辑"按钮
2. Modal标题显示"编辑监听规则"
3. 表单自动填充当前数据
4. 修改需要更改的字段
5. 保存时更新原记录（保持ID不变）
6. 显示"监听规则更新成功"提示

#### 2.3 配置转发策略流程
1. 点击监听规则的"配置转发策略"按钮
2. 打开转发策略配置Modal
3. Modal标题显示关联的监听规则名称
4. 配置域名组和路径规则：
   - 填写域名（必填）
   - 填写URL路径（可选，为空表示所有路径）
   - 选择虚拟服务器组（必填）
   - 填写备注（可选）
5. 支持操作：
   - 添加域名：新增域名组
   - 添加规则：在域名组内添加路径规则
   - 删除规则：删除特定路径规则
   - 删除域名组：删除整个域名组
6. 实时验证：
   - 域名格式验证（红框+错误提示）
   - URL格式验证（红框+错误提示）
   - "添加到列表"按钮状态控制（有错误时红色禁用）
7. 点击"添加到列表"：
   - 验证所有域名已填写
   - 验证域名格式正确
   - 验证每个域名至少有一个服务器组
   - 验证URL格式（如果填写）
8. 验证通过后批量添加到转发规则列表
9. 显示成功提示（已添加 N 条转发规则）
10. 表单重置

#### 2.4 删除监听规则流程
1. 点击"删除"按钮
2. 弹出确认对话框
3. 确认后从列表中移除
4. 相关联的转发策略同时删除

### 3. 虚拟服务器组管理

#### 3.1 新增服务器组流程
1. 点击"新增服务器组"按钮
2. 进入全页面创建界面
3. **步骤1：** 填写服务器组名称
   - 实时验证格式
   - 显示字符计数
4. **步骤2：** 选择虚拟机（表格多选）
   - 至少选择1台
   - 支持多选
5. **步骤3：** 自动显示配置区域（无需点击"下一步"）
   - 每台选中的虚拟机自动显示配置卡片
   - 默认端口8080，权重100
   - 支持添加多个端口
   - 支持删除端口（至少保留1个）
   - 支持移除虚拟机
6. 点击"确定创建"：
   - 验证服务器组名称格式
   - 验证至少有1台虚拟机
   - 验证每台虚拟机至少有1个端口配置
7. 生成唯一ID（格式：`sg-` + 时间戳）
8. 计算服务器数量
9. 默认健康检查为启用
10. 保存到服务器组列表
11. 返回详情页
12. 显示成功提示

#### 3.2 编辑服务器组流程
1. 点击操作列的"编辑"按钮
2. 进入全页面编辑界面
3. 页面标题显示"编辑虚拟服务器组"
4. 自动填充当前数据：
   - 服务器组名称
   - 已选中的虚拟机（表格自动勾选）
   - 每台虚拟机的端口和权重配置
5. 修改需要更改的内容
6. 点击"确定创建"
7. 更新原记录（保持ID不变）
8. 重新计算服务器数量
9. 返回详情页
10. 显示"虚拟服务器组更新成功"提示

#### 3.3 删除服务器组流程
1. 点击"删除"按钮
2. 弹出确认对话框
3. 确认后从列表中移除

**注意：** 实际生产环境应检查是否有监听规则或转发策略使用该服务器组

### 4. 证书管理

#### 4.1 上传证书流程
1. 点击"上传证书"按钮
2. 填写证书名称
3. 选择证书类型：
   - **服务器证书**：需要填写公钥证书和私钥
   - **CA证书**：需要填写客户端CA公钥证书
4. 粘贴证书内容（PEM格式）
5. 格式验证：
   - 公钥证书必须以`-----BEGIN CERTIFICATE-----`开头
   - 私钥必须以`-----BEGIN PRIVATE KEY-----`或`-----BEGIN RSA PRIVATE KEY-----`开头
6. 生成唯一ID（格式：`cert-` + 时间戳）
7. 记录创建时间
8. 初始关联监听为空数组
9. 保存到证书列表
10. 显示成功提示

#### 4.2 删除证书流程
1. 检查证书是否有关联监听规则
2. 如果有关联：
   - 删除按钮显示为禁用状态
   - 点击时提示："该证书已关联监听规则，无法删除"
3. 如果无关联：
   - 点击"删除"按钮
   - 弹出确认对话框
   - 确认后从列表中移除
   - 显示删除成功提示

### 5. 数据关联逻辑

#### 5.1 监听规则 → SSL证书
- 监听规则选择HTTPS协议时必须关联证书
- 证书被监听规则使用时不可删除
- 证书列表显示关联的监听规则名称

#### 5.2 监听规则 → 虚拟服务器组
- 每个监听规则必须关联一个虚拟服务器组
- 选择服务器组后自动显示组内虚拟机信息
- 服务器组被使用时显示关联的监听规则

#### 5.3 转发策略 → 监听规则
- 转发策略从属于监听规则
- 每条转发策略都有listenerId字段关联到监听规则
- 配置转发策略时自动关联到当前监听规则

#### 5.4 转发策略 → 虚拟服务器组
- 每条转发规则必须关联一个虚拟服务器组
- 服务器组被转发策略使用时显示关联的策略名称
- 支持不同路径转发到不同服务器组

#### 5.5 虚拟服务器组 → 虚拟机
- 服务器组包含多台虚拟机
- 每台虚拟机可以配置多个端口
- 每个端口配置对应不同的权重
- 虚拟机信息包含：ID、名称、内网IP

## 业务限制条件

### 1. 命名规则限制

| 资源类型 | 长度限制 | 字符限制 | 格式要求 |
|---------|---------|---------|---------|
| 负载均衡名称 | 无特殊限制 | 无特殊限制 | 无 |
| 监听规则名称 | 无长度限制 | 字母、数字、下划线 | 必须以字母或下划线开头 |
| 服务器组名称 | 1-80字符 | 中文、字母、数字、`-/.\_` | 无 |
| 证书名称 | 2-128字符 | 无特殊限制 | 无 |
| 域名 | 无明确限制 | 字母、数字、点号、星号 | 符合域名规范 |
| URL路径 | 2-80字符 | 字母、数字、`-/.%?#&` | 必须以`/`开头，不能只为`/` |

### 2. 数值范围限制

| 字段名称 | 最小值 | 最大值 | 类型 | 备注 |
|---------|-------|-------|------|------|
| 监听端口 | 1 | 65535 | 整数 | 标准端口范围 |
| 服务器端口 | 1 | 65535 | 整数 | 标准端口范围 |
| 权重 | 0 | 100 | 整数 | 0表示不分配流量 |

### 3. 数量限制

| 项目 | 限制 | 说明 |
|-----|------|------|
| 虚拟机选择 | 至少1台 | 服务器组必须包含至少一台虚拟机 |
| 端口配置 | 至少1个/虚拟机 | 每台虚拟机至少保留一个端口配置 |
| 域名组 | 至少1个 | 转发策略至少配置一个域名 |
| 路径规则 | 至少1个/域名 | 每个域名至少关联一个服务器组 |

### 4. 协议关联限制

| 协议类型 | 必填字段 | 限制说明 |
|---------|---------|---------|
| HTTP | 无特殊要求 | 不需要SSL证书 |
| HTTPS | SSL证书（必填） | 必须选择已上传的证书 |
| TCP | 无特殊要求 | 不需要SSL证书 |
| UDP | 无特殊要求 | 不需要SSL证书 |

### 5. 删除限制

| 资源类型 | 删除限制条件 | 提示信息 |
|---------|------------|---------|
| SSL证书 | 有关联的监听规则时不可删除 | "该证书已关联监听规则，无法删除" |
| 虚拟服务器组 | 建议检查是否有监听规则或转发策略使用 | 需要二次确认 |
| 监听规则 | 无特殊限制 | 删除时同时删除关联的转发策略 |
| 负载均衡实例 | 建议检查是否有关联资源 | 需要二次确认 |

### 6. 格式验证规则

#### 6.1 监听规则名称
- **正则表达式：** `/^[a-zA-Z_][a-zA-Z0-9_]*$/`
- **规则说明：**
  - 只能包含字母、数字、下划线
  - 必须以字母或下划线开头
- **错误示例：**
  - `123listener` ❌ (数字开头)
  - `listener-01` ❌ (包含连字符)
- **正确示例：**
  - `HTTP_Listener_80` ✅
  - `listener_001` ✅

#### 6.2 域名格式
- **正则表达式：** `/^(\*\.[\w-]+\.[\w-]+|[\w-]+\.[\w-]+(\.[a-z]+)*)$/`
- **规则说明：**
  - 支持泛解析域名（如`*.test.com`）
  - 支持标准域名（如`www.test.com`）
  - * 必须在最前面且后接点号
- **错误示例：**
  - `test` ❌ (缺少顶级域名)
  - `www.*test.com` ❌ (* 位置错误)
- **正确示例：**
  - `*.example.com` ✅
  - `www.example.com` ✅
  - `api.example.com.cn` ✅

#### 6.3 URL路径格式
- **正则表达式：** `/^\/[a-zA-Z0-9\-\/.%?#&]{1,79}$/`
- **规则说明：**
  - 长度2-80字符（含开头的`/`）
  - 必须以`/`开头
  - 不能只为`/`
  - 支持字符：字母、数字、`-/.%?#&`
- **错误示例：**
  - `api` ❌ (未以/开头)
  - `/` ❌ (只有单个斜杠)
  - `/api/测试` ❌ (包含中文)
- **正确示例：**
  - `/api` ✅
  - `/api/*` ✅
  - `/api/v1/users` ✅
  - `/static/*.css` ✅

#### 6.4 服务器组名称格式
- **正则表达式：** `/^[\u4e00-\u9fa5a-zA-Z0-9\-\/.\\_]+$/`
- **规则说明：**
  - 长度1-80字符
  - 支持中文、字母、数字
  - 支持特殊字符：`-/.\_`
- **错误示例：**
  - `` ❌ (空字符串)
  - `服务器组@01` ❌ (包含@符号)
- **正确示例：**
  - `web-server-group` ✅
  - `服务器组_01` ✅
  - `api/server/group.v1` ✅

#### 6.5 证书格式
- **公钥证书格式：** 必须以`-----BEGIN CERTIFICATE-----`开头
- **私钥格式：** 必须以`-----BEGIN PRIVATE KEY-----`或`-----BEGIN RSA PRIVATE KEY-----`开头
- **CA证书格式：** 必须以`-----BEGIN CERTIFICATE-----`开头
- **编码格式：** PEM格式

### 7. 表单验证时机

| 验证类型 | 验证时机 | 反馈方式 |
|---------|---------|---------|
| 格式验证 | 实时（输入时） | 红色边框 + 下方错误提示 |

### 关联关系图

```
负载均衡实例 (LoadBalancer)
    │
    ├── 监听规则 (ListenerRule) [1:N]
    │   │
    │   ├── SSL证书 (Certificate) [N:1] (仅HTTPS)
    │   │
    │   ├── 虚拟服务器组 (ServerGroup) [N:1]
    │   │
    │   └── 转发策略 (ForwardingPolicy) [1:N]
    │       │
    │       └── 虚拟服务器组 (ServerGroup) [N:1]
    │
    └── 虚拟服务器组 (ServerGroup) [1:N]
        │
        └── 虚拟机 (VMInstance) [N:M]
            └── 端口配置 (PortConfig) [1:N]
```

### 数据模型

#### LoadBalancer（负载均衡实例）
```typescript
{
  id: string                 // 唯一标识
  name: string              // 名称
  listenerCount: number     // 监听规则数量
  serverGroupName: string   // 主要服务器组名称
}
```

#### ListenerRule（监听规则）
```typescript
{
  id: string                           // 唯一标识
  name: string                         // 规则名称
  protocol: 'HTTP'|'HTTPS'|'TCP'|'UDP' // 协议
  port: number                         // 端口
  certificate?: string                 // SSL证书ID（HTTPS时）
  serverGroup: string                  // 服务器组名称
  healthCheck: 'enabled'|'disabled'    // 健康检查
  status: 'active'|'inactive'          // 状态
}
```

#### ForwardingPolicy（转发策略）
```typescript
{
  id: string          // 唯一标识
  listenerId: string  // 关联的监听规则ID
  domain: string      // 域名
  path: string        // URL路径
  serverGroup: string // 虚拟服务器组名称
  remark?: string     // 备注
}
```

#### ServerGroup（虚拟服务器组）
```typescript
{
  id: string                          // 唯一标识
  name: string                        // 组名称
  serverCount: number                 // 服务器数量
  healthCheck: 'enabled'|'disabled'   // 健康检查
  servers: ServerConfig[]             // 服务器配置列表
}
```

#### ServerConfig（服务器配置）
```typescript
{
  vmId: string           // 虚拟机ID
  vmName: string         // 虚拟机名称
  privateIp: string      // 内网IP
  ports: PortConfig[]    // 端口配置列表
}
```

#### PortConfig（端口配置）
```typescript
{
  id: string      // 唯一标识
  port: number    // 端口号
  weight: number  // 权重
}
```

#### Certificate（SSL证书）
```typescript
{
  id: string                // 唯一标识
  name: string              // 证书名称
  createTime: string        // 创建时间
  relatedListeners: string[] // 关联的监听规则名称列表
  type: 'server'|'ca'       // 证书类型
  publicKey?: string        // 公钥证书（服务器证书）
  privateKey?: string       // 私钥（服务器证书）
  caPublicKey?: string      // CA公钥证书（CA证书）
}
```

### B. 常见使用场景

#### B.1 场景1：简单HTTP负载均衡
1. 创建负载均衡实例 `web-lb-prod`
2. 创建服务器组 `web-server-group`（包含3台Web服务器）
3. 创建HTTP监听规则（端口80）关联到 `web-server-group`
4. 流量自动分发到3台服务器

#### B.2 场景2：HTTPS + 域名路由
1. 创建负载均衡实例 `api-lb-prod`
2. 上传SSL证书 `api.example.com`
3. 创建服务器组 `api-server-group` 和 `static-server-group`
4. 创建HTTPS监听规则（端口443）关联 `api-server-group`
5. 配置转发策略：
   - 域名 `api.example.com`，路径 `/api/*` → `api-server-group`
   - 域名 `api.example.com`，路径 `/static/*` → `static-server-group`
6. 不同路径的流量分发到不同服务器组

#### B.3 场景3：多端口服务
1. 创建负载均衡实例 `service-lb-prod`
2. 创建服务器组 `multi-service-group`
3. 配置虚拟机：
   - VM1：端口8080(权重100)、端口8081(权重50)
   - VM2：端口8080(权重100)、端口8081(权重50)
4. 创建多个监听规则分别监听不同端口
5. 支持同一服务器提供多种服务

### C. 最佳实践建议

1. **命名规范**
   - 使用有意义的名称，包含环境标识（如 `prod`、`test`）
   - 统一命名风格（如使用连字符或下划线）

2. **安全性**
   - HTTPS监听规则必须使用SSL证书
   - 定期更新证书（注意过期时间）
   - 使用CA证书时确保客户端证书有效

3. **高可用性**
   - 服务器组至少包含2台虚拟机
   - 启用健康检查，及时发现故障服务器
   - 合理配置权重，避免单点过载

4. **性能优化**
   - 根据服务器性能调整权重值
   - 权重为0的服务器不接收流量（可用于维护）
   - 同一服务器的不同端口可配置不同权重

5. **转发策略设计**
   - 使用泛域名支持子域名通配
   - URL路径支持通配符（如 `/api/*`）
   - 空URL路径匹配所有路径（默认路由）
   - 合理规划域名和路径，避免冲突
