# 2. 需求概述

## 2.1 需求简介

本需求文档描述当前前端原型中已实现的两类能力：`环境下线` 与 `资源盘点`，覆盖入口、确认、执行进度、失败重试与结果展示。  
原型目标是让产品/前端/后端在同一语义下评审下线流程，不引入当前代码中不存在的审批流、工单流与批量任务。

**当前原型范围（已实现）**

1. 在管理台-游戏管理中，进入某个 `appId` 的环境详情页。
2. 在“游戏初始化”模块中，对测试或正式环境发起“下线环境”操作。
3. 展示下线确认弹窗、下线进度弹窗、资源级状态与失败重试。
4. 下线成功后，自动把该环境客户端/服务端状态置为 `not_initialized`。
5. 在游戏列表中对单个 `appId` 发起“资源盘点”，按测试/正式环境输出资源汇总与明细。

**非目标（当前原型未实现）**

- 审批流（申请、审批人、驳回）。
- 定时下线、批量下线。
- 真正后端 API 调用（当前是前端 mock + 模拟异步删除）。
- 细粒度权限控制。

## 2.2 预计上线时间

- 原型已可运行（当前分支）
- 文档评审后按开发排期进入接口联调阶段（待定）

## 2.3 风险点

1. 当前“删除成功率”采用随机模拟，评审时容易与真实后端能力混淆。
2. “后台运行”仅关闭弹窗，不是独立后台任务系统。
3. 下线完成后仅更新初始化状态，未联动其他模块（公告、通知、权限等）。
4. 若未来改为真实 API，需补齐幂等与重入处理。

## 2.4 设计稿

- 当前以代码原型为准，暂无独立 Figma 链接。

# 3. 功能详情

## 3.1 入口与触发条件

### 3.1.1 入口路径

- 路径：`/admin` -> 左侧菜单“游戏管理” -> 列表点击 `App ID` -> 进入环境详情页。
- 环境下线入口位于“游戏初始化”卡片内，按环境（测试/正式）分别展示。

### 3.1.2 按钮显示规则

**业务规则**

- 仅当环境满足“客户端或服务端任一已初始化”时，显示“下线环境”按钮。
- 若测试和正式环境都未初始化，则不展示下线操作区。

**条件字段（来自原型状态）**

| 字段名 | 类型 | 默认值 | 取值范围/枚举 | 校验规则 |
|---|---|---|---|---|
| `clientStatus` | enum | `not_initialized` | `initialized` / `not_initialized` | 用于判断是否展示下线入口 |
| `serverStatus` | enum | `not_initialized` | `initialized` / `not_initialized` | 用于判断是否展示下线入口 |

## 3.2 下线确认交互

### 3.2.1 确认弹层

点击“下线环境”后弹出二次确认，文案明确提示：

- 下线将删除该环境所有相关资源（云服务、存储、网络配置等）
- 操作不可撤销

**操作项**

- 确认下线（danger）
- 取消

### 3.2.2 输入参数（当前原型）

| 字段名 | 类型 | 默认值 | 取值范围/枚举 | 校验规则 |
|---|---|---|---|---|
| `appId` | string | 当前详情页 `appId` | 详情页上下文值 | 必填 |
| `environment` | enum | 无 | `test` / `prod` | 必填 |
| `deployType` | enum | 来自环境初始化配置 | `vm` / `container` / `''` | 用于计算资源清单 |

## 3.3 下线执行与进度展示

### 3.3.1 执行模型

确认后立即启动模拟异步删除流程，并打开“环境下线进度”弹窗。

**业务规则**

1. 资源按列表顺序逐个处理。
2. 单个资源状态流转：`pending -> deleting -> completed/failed`。
3. 首次删除成功率约 90%，失败后可手动重试。
4. 重试成功率约 85%（仍可能失败）。
5. 整体进度用 `currentStep / totalSteps` 展示，并同步进度条。

### 3.3.2 资源清单生成规则

根据 `environment + deployType` 动态生成资源列表：

- 公共资源：AWS + 阿里云基础资源（IAM/CloudFront/S3/Route53、RAM/Tair/ACR/PolarDB/MongoDB/MSE/OSS/GA 等）
- `vm` 部署追加：ECS、CLB
- `container` 部署追加：K8s Deployment/HPA/CronJob/Job/Service/Ingress/ConfigMap/PVC/PV/Namespace + 平台内部数据
- 正式环境通常比测试环境资源数量更大（如冗余与备份资源）

### 3.3.3 列表展示字段

- 资源名称
- 资源类型
- 分类（AWS/AliCloud/Kubernetes/平台内部数据）
- 数量
- 示例明细（最多展示前 N 条）
- 当前状态（等待中/删除中/已完成/删除失败）
- 失败态操作（重试）

### 3.3.4 自动配置默认值

| 字段名 | 类型 | 默认值 | 取值范围/枚举 | 校验规则 |
|---|---|---|---|---|
| `offlineProgress.currentStep` | number | `0` | `0 ~ totalSteps` | 执行中递增 |
| `offlineProgress.totalSteps` | number | 资源数量求和 | `>= 0` | 初始化时计算 |
| `resource.status` | enum | `pending` | `pending/deleting/completed/failed` | 按执行过程更新 |

## 3.4 完成与失败处理

### 3.4.1 下线完成

当全部资源完成删除后：

1. `isOfflineInProgress` 置为 `false`
2. 当前环境初始化状态重置为未初始化
   - `clientStatus = not_initialized`
   - `serverStatus = not_initialized`
3. 展示成功提示
4. 延时关闭进度弹窗并清理进度状态

### 3.4.2 失败与重试

**业务规则**

- 任一资源删除失败时，不自动跳过，停留失败态并提示错误。
- 用户可对失败资源点击“重试”。
- 当重试后全部资源均为 `completed`，进入成功收敛流程。

## 3.5 交互约束（当前原型）

### 3.5.1 进度弹窗行为

- 执行中：允许点击“后台运行”关闭弹窗。
- 执行中：不允许直接点右上角关闭（`closable=false`）。
- 弹窗遮罩不可点击关闭（`maskClosable=false`）。
- 执行完成：显示“关闭”按钮。

### 3.5.2 列表与详情关系

- 游戏列表中通过点击 `App ID` 进入详情（不是“详情”按钮）。
- 下线操作只在详情页内进行，不在列表直接触发。

## 3.6 资源盘点

### 3.6.1 入口与触发

- 入口位置：`/admin` -> 游戏管理列表 -> “操作”列点击“盘点”按钮。
- 触发后打开“资源盘点”弹窗，默认环境为 `test`。
- 用户可切换环境（`test` / `prod`）后点击“开始盘点”。

### 3.6.2 输入参数与默认值

| 字段名 | 类型 | 默认值 | 取值范围/枚举 | 校验规则 |
|---|---|---|---|---|
| `appId` | string | 当前行 `appId` | 列表上下文值 | 必填 |
| `environment` | enum | `test` | `test` / `prod` | 必填 |
| `deployType` | enum | 来自该环境配置 | `vm` / `container` / `''` | 可为空（待配置） |
| `isInventoryLoading` | boolean | `false` | `true/false` | 点击“开始盘点”后置为 `true` |

### 3.6.3 执行逻辑（当前原型）

**业务规则**

1. 点击“开始盘点”后进入 loading 态，模拟盘点耗时约 2 秒。
2. 根据 `appId + environment + deployType` 生成资源列表与数量汇总。
3. 盘点完成后展示成功提示与结果面板。
4. 关闭弹窗会清理本次盘点上下文（选中游戏、结果、loading 状态）。

### 3.6.4 结果展示字段

- 环境维度总资源数（`totalCount`）
- 分类汇总（AWS / AliCloud / Kubernetes）：
  - 每个分类的实例总数
  - 每个分类的资源类型数
- 资源明细：
  - 资源名
  - 分类
  - 类型
  - 实例数量
  - 示例实例名（超出部分折叠为“等 N 个实例”）

### 3.6.5 与下线能力的关系

- 盘点与下线共用相似资源模型（按环境、部署类型动态生成资源）。
- 盘点是“只读统计”，不改变游戏状态。
- 下线是“写操作模拟”，会改写环境初始化状态为未初始化。

# 4. 异常情况处理

## 4.1 下线异常处理

- 重复提交：前端禁用确认按钮；后端用 `appId + environment + requestId` 做幂等。
- 页面刷新/离开：以 `taskId` 恢复任务进度；存在进行中任务时禁止重复发起。
- 资源删除失败：展示失败项并支持重试；仅重试失败资源。
- 多次重试失败：重试上限建议 3 次，超限标记“人工介入”。
- 超时/断网：前端提示“状态未知，请刷新”；后端任务持续执行并可查询结果。
- 并发下线：同一 `appId + environment` 加互斥锁，只允许一个进行中任务。

## 4.2 资源盘点异常处理

- 盘点中切换环境：执行中锁定环境；切换需先取消再重开。
- 重复发起盘点：执行中禁用“开始盘点”；若并发，需按 `taskId` 区分结果。
- 盘点超时：建议 30 秒超时并提示“可重试”。
- 汇总不一致：`totalCount` 与明细求和不一致时，标记盘点失败并阻止展示错误结果。

## 4.3 通用兜底

- 错误分级：参数错误（引导修正）、状态错误（说明不可操作原因）、系统错误（提示稍后重试）。
- 最小日志：`taskId`、`appId`、`environment`、操作人、开始/结束时间、结果状态、错误码。
- 展示原则：失败必须可见，成功必须基于真实状态返回。
