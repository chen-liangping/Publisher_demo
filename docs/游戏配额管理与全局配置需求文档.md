# 3. 功能详情

## 3.1 游戏配额管理

### 3.1.1 配置范围

* 按环境维护两套配额：`test` / `prod`
* 两个环境相互独立保存
* 根据部署类型展示不同配额字段：
  * 容器部署：容器配额字段
  * 虚机部署：虚机配额字段

### 3.1.2 容器部署可配字段

| 字段名 | 类型 | 约束 |
|-----|------|------|
| `cpuCores` | number | 1-64 |
| `maxPodReplicasPerPod` | number | 1-1000 |
| `maxAppsPerGame` | number | 1-10000 |
| `maxCronJobsPerGame` | number | 1-10000 |
| `maxImagesPerGame` | number | 1-10000 |

### 3.1.3 虚机部署可配字段

| 字段名 | 类型 | 约束 |
|-----|------|------|
| `vmCpuCores` | number | 1-128 |
| `memoryGB` | number | 1-2048 |
| `vmDiskTotalSizeGB` | number | 1-65536 |

### 3.1.4 校验与保存规则

* 所有字段通过表单校验后才可提交
* 新值不得小于已使用值（否则阻断提交）
* 区间字段要求 `max >= min`
  * Redis 分片范围
  * Mongo Mongos/Mongod/Config 范围
* 点击确认后走二次确认，再执行保存

## 3.2 全局配置

### 3.2.1 全局监控配置（全局生效）

| 字段名 | 类型 | 默认值 | 说明 |
|-----|------|------|------|
| `grafanaConfig` | boolean | false | Grafana 配置完成状态 |
| `cdnConfig` | boolean | false | CDN 配置完成状态 |

**业务规则**

* 点击“完成配置”后弹出确认框
* 确认后字段置为 `true`
* 页面立即刷新状态，显示“已配置”
* 当前不支持撤销为 `false`

### 3.2.2 资源全局配置（按环境 test/prod）

#### 基础策略字段

| 字段名 | 类型 | 默认值 | 约束 |
|-----|------|------|------|
| `storageInstancesLimit` | number | 5 | 0-100 |
| `storageBackupGames` | string[] | `['shinchan']` | 多选（prod 使用） |
| `vmAllowedSpecs` | string[] | `['2核4G','2核8G','4核8G']` | 多选 |

#### MySQL 字段

| 字段名 | 类型 | 默认值 |
|-----|------|------|
| `mysqlVersions` | string[] | `['5.7','8.0']` |
| `mysqlSpecs` | string[] | `['2核8G']` |

#### Redis 字段

| 字段名 | 类型 | 默认值 | 约束 |
|-----|------|------|------|
| `redisMasterSlaveVersions` | string[] | `['5.0','6.0','7.0']` | 多选 |
| `redisMasterSlaveSpecs` | string[] | `['1GB']` | 多选 |
| `redisShardVersions` | string[] | `['5.0','6.0','7.0']` | 多选 |
| `redisShardSpecs` | string[] | `['1GB']` | 多选 |
| `redisShardCountMin` | number | 2 | 1-128 |
| `redisShardCountMax` | number | 3 | 1-128 且 `>=Min` |

#### MongoDB 字段

| 字段名 | 类型 | 默认值 | 约束 |
|-----|------|------|------|
| `mongoReplicaVersions` | string[] | `['5.0']` | 多选 |
| `mongoReplicaSpecs` | string[] | `['2核8G']` | 多选 |
| `mongoShardedVersions` | string[] | `['5.0']` | 多选 |
| `mongoShardMongosCountMin` | number | 2 | 0-20 |
| `mongoShardMongosCountMax` | number | 2 | 0-20 且 `>=Min` |
| `mongoShardMongosSpecs` | string[] | `['2核8G']` | 多选 |
| `mongoShardMongodCountMin` | number | 2 | 0-20 |
| `mongoShardMongodCountMax` | number | 2 | 0-20 且 `>=Min` |
| `mongoShardMongodSpecs` | string[] | `['2核8G']` | 多选 |
| `mongoShardConfigCountMin` | number | 1 | 0-20 |
| `mongoShardConfigCountMax` | number | 1 | 0-20 且 `>=Min` |
| `mongoShardConfigSpecs` | string[] | `['4核8G']` | 多选 |

#### Zookeeper 字段

| 字段名 | 类型 | 默认值 | 约束 |
|-----|------|------|------|
| `zookeeperVersions` | string[] | `['3.8']` | 多选（3.8/3.9） |
| `zookeeperSpecs` | string[] | `['1核2G']` | 多选 |
| `zookeeperNodeCount` | number | 1 | 1-9 |

## 3.3 业务规则补充

* 全局监控配置与资源全局配置属于不同模块，需在页面上明确区分命名
* 字段默认值在 test/prod 可不同，保存后即时生效
* 资源配置中的区间字段必须进行前端校验，避免脏数据写入
