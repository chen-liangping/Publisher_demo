# 游戏通知配置需求文档

## 1. 产品概述

### 1.1 功能定位
游戏通知配置页面是 Publisher 平台管理台的核心功能模块，用于统一管理游戏运营过程中的各类系统通知规则，确保关键事件能够及时、准确地推送给相关人员。

### 1.2 业务价值
- **提升运营效率**：自动化通知机制减少人工监控成本
- **降低运营风险**：关键事件及时通知，避免问题扩大
- **优化协作流程**：不同类型通知分发到对应群组，提高响应效率

## 2. 功能需求

### 2.1 页面结构
- **入口**：管理台 → 游戏通知配置
- **布局**：通知渠道配置 + 消息通知配置（两个卡片）

### 2.2 通知渠道
| 渠道 | 用途 |
|------|------|
| Publisher大群 | 全局业务通知 |
| monitor 群 | 监控告警同步 |

### 2.3 通知规则配置
- **搜索**：支持主题/类型/ID模糊搜索
- **表格列**：通知主题、通知类型、接收渠道
- **交互**：复选框控制各渠道启用状态，实时生效

## 3. 数据模型

### 3.1 通知主题
- **HPA**：扩缩容事件
- **静态资源与CDN**：资源同步事件  
- **运行时健康**：服务监控事件

### 3.2 预设规则
| 主题 | 类型 | 默认渠道 |
|------|------|---------|
| HPA | 开启/关闭HPA | 全部渠道 |
| 静态资源与CDN | 翻译文本同步CDN失败 | 全部渠道 |

## 5. 技术实现

### 5.1 技术栈
- **前端框架**：Next.js (App Router)
- **UI组件库**：Ant Design 5.x
- **状态管理**：React useState Hook
- **类型定义**：TypeScript

### 5.2 核心组件
```typescript
// 主组件
GlobalNoticeConfig: React.ReactElement

// 状态管理
const [rules, setRules] = useState<NoticeRuleRow[]>()
const [keyword, setKeyword] = useState<string>()

// 核心方法
toggleChannel(rowId: string, channelKey: ChannelKey): void
```

### 5.3 数据流
1. **初始化**：加载预设通知规则
2. **搜索过滤**：根据关键词实时过滤规则列表
3. **配置变更**：点击复选框更新规则状态
4. **状态持久化**：变更立即保存到组件状态

## 6. UI/UX 设计要求

### 6.1 视觉风格
- **整体风格**：简洁现代，符合管理台设计语言
- **色彩方案**：主色调与平台保持一致
- **圆角设计**：卡片和输入框使用圆角设计

### 6.2 布局规范
- **间距标准**：卡片间距 14px，内容间距 12px
- **字体规范**：标题 700 字重，描述文字 400 字重
- **响应式**：支持不同屏幕尺寸自适应

### 6.3 交互反馈
- **悬停效果**：按钮和可点击元素提供悬停反馈
- **状态指示**：复选框清晰显示选中/未选中状态
- **操作确认**：配置变更提供视觉确认

## 7. 验收标准

### 7.1 功能验收
- [ ] 页面正常加载，显示所有预设通知规则

- [ ] 表格支持横向滚动，内容完整显示

## 8. 后续扩展

- **批量操作**：支持批量启用/禁用渠道
