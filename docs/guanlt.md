
# 游戏初始化

下面把「游戏管理 → 新增游戏 → 初始化配置」这一整条交互链路中**所有弹窗、Toast、文字提示**梳理成一张「**交互清单**」。  
已按出现顺序归类，方便后续由前端、QA、运营直接对照实现与验收。  

---

|---|---|---|---|
| 点击【添加游戏】按钮 | 弹窗标题 | 添加游戏 | 一级弹窗 |
|  | 输入框① | APP ID<br>占位符：请输入appid | 必填，校验规则见下 |
|  | 输入框② | 描述<br>占位符：请输入游戏描述 | 非必填 |
|  | 开关① | 客户端资源 | 默认关；开启后出现“已配置”字样 |
|  | 开关② | 服务端资源 | 默认关；开启后出现“已配置”字样 |
|  | 开关③ | 全球加速<br>副文案：启用全球加速 | 默认关 |
|  | 开关④ | flashlaunch<br>副文案：启用flashlaunch | 默认关 |

## 游戏管理业务规则

**特殊处理**

* 删除APP ID
  * 未进行初始化，可执行删除操作
  * 已初始化的APP，删除操作置灰
* 修改客户端/服务端配置
  * 已初始化：点击toast提示“已完成初始化，不可修改”
  * 未初始化：点击弹窗确认是否修改配置，确认后完成修改
* 全球加速
  * 开启：点击toast提示“开启后无法关闭”
  * 关闭：点击弹窗确认是否开启，确认后完成修改
* flashlaunch
  * 点击弹窗确认是否修改配置，确认后完成修改

  

#### 添加游戏


1. 操作入口：admin管理页面-平台游戏
2. UI交互：

   
   1. 点击”添加“icon，出现添加游戏弹窗，输入以下内容

      
      1. APP ID：必填
      2. 描述：非必填，针对游戏的描述说明
      3. 客户端资源：**切换开关**，默认打开，（已勾选之后且初始化成功后不可取消）
      4. 服务端资源：**切换开关**，默认打开，（已勾选之后且初始化成功后不可取消）
      5. 全球加速：勾选项，默认不勾选（开启后不可关闭）==已确认==
      6. flashlaunch：勾选项，默认勾选（初始化后可修改）
   2. 可选操作：

      
      1. 确定：保存配置
      2. 关闭：退出配置界面

**添加游戏成功的必要条件：服务端和客户端资源至少配置一项**

**未配置客户端的情况下，不可开启全球加速和flashlaunch，提示“未配置客户端资源、无法开启”**

#### 删除游戏

对于完全未进行初始化的游戏，可执行”删除“操作，


1. 操作入口：admin管理页面-平台游戏
2. UI交互：

   
   1. 点击”删除“，弹出确认框提示”删除后数据不可恢复，是否删除“， 

      
      1. 可选操作：

         
         1. 确定：删除游戏
         2. 关闭：退出当前确认界面
      2. 游戏删除后CP不可在publisher 页面搜索到该ID


### 4.2 publisher 游戏初始化



1. 操作入口：平台游戏列表页-初始化
2. UI交互：

   
   1. 搜索APPID，根据admin选择的客户端和服务端配置，分模块初始化
   2. 客户端 展示“S3 和CDN 尚未初始化，请点击初始化按钮执行操作”
   3. 服务端展示“K8S 配置尚未初始化，请点击初始化按钮执行操作”
   4. 点击执行初始化操作后，展示初始化进度条


**说明**

* 游戏初始化为异步操作，关闭当前初始化窗口后再次打开，仍展示初始化进度
* 若初始化失败，进度条可点击“重试”按钮，从失败位置重新初始化
* 日志记录 cp 初始化操作

可直接用于 PRD、前端任务单或 QA 用例。

| 区域/元素 | 文案/交互 | 备注 |

| 搜索框 | 请输入 appid | 实时过滤，支持模糊匹配 |
| 列表表头 | appid / 应用资源CPU核数 / MySQL实例数量 / Mongo实例数量 / Redis实例数量 / Zookeeper实例数量 / 操作 | 固定顺序 |

## 4.1 游戏管理


### 4.1.1 添加游戏


1. 操作入口：admin管理页面-平台游戏
2. UI交互：

   
   1. 点击「新增游戏」
   2. 输入 appId 并提交
   3. 新增成功后，列表立即出现该 appId（初始化状态默认为「待初始化」）

添加完成后，可在详情页查看app配置


**业务规则**

* 服务端和客户端资源至少配置一项
* 未配置客户端的情况下，不可开启全球加速和flashlaunch，hover提示“未配置客户端资源、无法开启”
* 游戏添加成功且未初始化，允许修改配置项；
* 初始化完成
  * 已配置的客户端和服务端不可取消配置；
  * 全球加速开启后不可关闭；
  * flashlaunch 支持修改
* 添加游戏时，配置项应用于测试环境和生产环境，添加完成后，生产环境配置项可单独修改
* **测试环境已初始化的配置项（客户端、服务端），生产环境不可取消配置**

* 新增时仅录入 appId，不要求同步配置测试/生产环境字段
* 新增后默认：
  * 测试环境初始化状态：待初始化
  * 生产环境初始化状态：待初始化
  * 服务端部署方式：为空（页面展示为「待配置」）
  * 全局监控（Grafana/CDN）：默认关闭

### 4.1.3 游戏列表

列表字段如下：

* APP ID：展示APP ID，最长24字符
* 部署方式：云原生、虚拟机、待配置（新增后为空时展示）
* 测试环境：待初始化、初始化完成
* 正式环境：待初始化、初始化完成
* 操作：下线、删除


### 4.1.3 游戏详情


初始化配置（当前原型交互）

* Grafana 配置与 CDN 配置为 appId 的全局配置，不区分测试环境和生产环境
  * 展示位置：初始化配置卡片顶部「全局监控」区域
  * 展示形式：两行开关样式（只读展示），点击「编辑全局监控」弹窗里可修改并保存
* 测试环境与生产环境的初始化数据（客户端/服务端状态、GA/FL）在同一张初始化配置卡片中上下排列展示
  * 每个环境块包含：
    * 客户端初始化状态（Tag：已完成 / 待初始化）
    * 服务端初始化状态（Tag：已完成 / 待初始化）
    * 全球加速（GA）开关（可点击，点击后弹二次确认）
    * FlashLaunch（FL）开关（可点击，点击后弹二次确认）
  * 环境块右上角提供「查看该环境」，用于切换详情页里其他配置卡片（限额/开服失败等）的展示环境
* 二次确认弹窗会明确提示本次修改属于「测试环境」或「生产环境」

appid录入完成后，可在详情页查看配置、编辑配置


部署方式（当前原型展示）

* 服务端部署方式为测试/生产共享字段（统一展示为一份）
* 新增 appId 时部署方式为空，页面展示为「待配置」



### 4.1.4 游戏下线


点击admin管理后台-游戏管理中的”下线“，出现二次确认弹窗，点击确定，执行下线操作

异步删除，需返回删除日志（grafana 日志），若中途失败，可从失败位置重试


**业务规则**

* 删除操作不可撤销
* 每删除一个资源（数据库表、存储桶、CDN 配置等）后，立即验证系统是否正常。
* 当某一步删除失败时，后续删除任务将暂停执行，直到前一步任务重试并成功完成后，才会继续执行下一步。
* **下线需要删除资源**
  * AWS
    * S3
    * IAM
    * CloudFront
  * AliCloud
    * OSS
    * RAM
    * ACR NS
    * Tair
  * grafana 日志数据
  * Prometheus 配置

 **[资源下线注意事项](https://w.g123.jp/doc/doneapp-CHRegVznTE)**


### 4.1.2 删除游戏



1. 操作入口：admin管理页面-平台游戏
2. UI交互：

   
   1. 点击”删除“，弹出确认框提示”删除后数据不可恢复，是否删除“， 

      
      1. 可选操作：

         
         1. 确定：删除游戏
         2. 关闭：退出当前确认界面

* 游戏删除后不可在publisher 页面搜索到该ID


**业务规则**

* 未初始化游戏支持删除操作
* 已下线游戏支持删除操作

## 4.2 publisher 游戏初始化



1. 操作入口：平台游戏列表页-初始化
2. UI交互：

   
   1. 搜索APPID，根据admin选择的客户端和服务端配置，分模块初始化
   2. 客户端 展示“客户端配置尚未初始化，请点击初始化按钮执行操作”
   3. 服务端展示“服务端配置尚未初始化，请点击初始化按钮执行操作”，点击后选择部署方式“容器部署、虚机部署”，确认部署方式，执行初始化
   4. 确认执行初始化操作，展示初始化进度条


**说明**

* 游戏初始化为异步操作，关闭当前初始化窗口后再次打开，仍展示初始化进度
* 若初始化失败，进度条可点击“重试”按钮，从失败位置重新初始化
* Publisher 操作日志记录 cp 初始化操作
