# Mongo 数据库权限与分片 — 业务要点（短版）

- 页面分别展示**只读账号**与**读写账号**（账号信息及权限范围）。
- 文案：**「创建数据库」** 改为 **「分配数据库权限」**；**「新建数据库」** 改为 **「新增数据库权限」**。
- 页面直接展示**每个账号**对**各数据库**的权限（有/无等）；并展示该库维度 **分片已开启 / 未开启**；可下钻或查询**各集合的分片状态**。

**分片状态**

- `未分片` | `已分片`。
- 开启后**不可关闭**（不提供「关闭分片」类能力）。

**分片键**

- 用于分片的字段名（如 `_id`、`userId` 等）。
- **已分片**：可**调整分片字段**（及类型，与产品/后端能力一致）。
- **未分片**：可**开启分片**，并**设置分片键**（及类型）。

**分片键类型**

- `hashed`、`range`（与 Mongo 侧 hashed / range 分片语义对应）。

**数据分布**

- 结论维度：**均衡** | **不均衡**（具体判定依赖管控/监控**是否返回**分布或 chunk 占比等字段；无数据则仅展示「未知」或不展示该项，由实现约定）。
